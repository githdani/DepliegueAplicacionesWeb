<h1 align="center">Filezilla - Uso del navegador como cliente FTP</h1>

## Acceder al servidor FTP desde un navegador.

Actualmente, los navegadores web modernos han eliminado el soporte para el ***protocolo FTP*** debido a motivos de **seguridad** y **mantenimiento**. El protocolo FTP transmite credenciales en texto plano y no cumple con los estándares de seguridad actuales, lo que llevó a que los principales navegadores dejaran de implementarlo.

Navegadores como ***Google Chrome***, ***Mozilla Firefox*** y ***Microsoft Edge*** ya no permiten conexiones FTP, independientemente de que el servidor esté correctamente configurado. Por este motivo, el uso del navegador como **cliente FTP** no es viable en entornos actuales y no depende de la configuración del servidor, sino de las limitaciones del propio navegador.

---

## Ventajas y desventajas.

| Tipo | Característica | Descripción |
|----|---------------|------------|
| **_Ventaja_** | No requiere instalación | Se accede directamente desde el navegador |
| **_Ventaja_** | Uso sencillo | Interfaz conocida para usuarios básicos |
| **_Desventaja_** | Soporte eliminado | Navegadores modernos no soportan FTP |
| **_Desventaja_** | No permite subir archivos | Generalmente solo lectura |
| **_Desventaja_** | No soporta FTPS | No permite conexiones cifradas |
| **_Desventaja_** | Sin control de modos | No permite configurar modo activo/pasivo |
| **_Desventaja_** | Baja seguridad | Credenciales expuestas o no soportadas |
